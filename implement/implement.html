02 — Structural Status

The desktop layout is mostly there. The two-column grid, nav bar, frosted glass card, and hover states are all working, and the BBH Hegarty font loads correctly. The hierarchy feels readable and all the links go where they should. Mobile is built but still rough around the edges. Spacing is tight and a few items clip on smaller screens. The Figma grouping into Sketches, Project, and Reflections translated well, and the gradient background looks consistent between the mockup and the browser. Where things break is in the details. The frosted glass flattens out in Safari, the right column feels a little sparse at wider viewport widths, and BBH Hegarty loses some of its character at small sizes in a way it did not in Figma.

03 — Translation Challenges

The biggest thing I did not expect was how much Figma was quietly doing for me. The frosted glass card felt like one decision in the mockup. In CSS it became four or five things that all need to work together, and it still behaves differently in Chrome versus Safari. More broadly, Figma holds things exactly where you put them. A browser flows. Every visual grouping that felt obvious in the frame had to be deliberately reconstructed in markup, and the mobile layout is where that gap showed up most. What looked clean in Figma had to be rebuilt from scratch rather than just scaled down.

04 — Next Steps

The main focus for the final week is the mobile layout. It needs real device testing, not just browser dev tools, and the spacing issues need fixing before submission. After that, swapping in real screenshots, writing proper descriptors for sketches 4 through 10, and a quick cross-browser check. Last thing is making sure all the assessment pages link correctly from the index.
